# 贡献指南 / Contributing

感谢你愿意帮助完善 SwiFRP！请在提交前阅读以下说明。

Thanks for considering contributing to SwiFRP! Please review the notes below before you start.

## 提交流程 / Workflow

1. 先在 Issue 中描述你的问题或建议（或在已有 Issue 下讨论）。
2. 按照 `feat/`、`fix/`、`chore/`、`refactor/` 规则创建分支。
3. 进行修改并确保通过 `swift test`。
4. 提交 PR，并简要说明变更内容与测试情况。

1. Discuss your idea in an issue first (or join an existing one).
2. Create a branch using `feat/`, `fix/`, `chore/`, or `refactor/` prefixes.
3. Make changes and ensure `swift test` passes.
4. Open a PR with a short summary and test results.

## 开发要求 / Development Requirements

- macOS 13.0+ / Swift 5.9+
- 推荐使用 Xcode 或 `swift` CLI 构建
- 请避免提交与任务无关的格式化变更

- macOS 13.0+ / Swift 5.9+
- Use Xcode or the `swift` CLI for builds
- Avoid unrelated formatting changes

## 提示 / Tips

- 涉及 UI 或交互变更时，请附上截图。
- 如果你新增了功能，请补充相应说明。

- Include screenshots for UI/UX changes.
- Update documentation if you add new features.
